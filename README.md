# Music Library

A single-user web application for managing a music catalog — artists, albums, songs, and playlists. Built to evaluate how well Deepseek-V4-Flash-REAP-180B handles a full project from scratch — planning, coding, organizing, and fixing — with no human intervention beyond prompts.

## Tech Stack

| Layer | Choice |
|---|---|
| **Backend** | Node.js + Express.js |
| **Database** | SQLite via built-in `node:sqlite` (Node ≥22) |
| **Frontend** | React 18+ with Vite |
| **Routing** | React Router v6 |
| **HTTP** | Native `fetch` |
| **Monorepo** | npm workspaces |

## Quick Start

```bash
npm install
npm run dev
```

Server runs on `http://localhost:3001`, client on `http://localhost:5173` (proxied to server).

## Scripts

| Command | Purpose |
|---|---|
| `npm run dev`      | Start both server and client |
| `npm run lint`     | ESLint across the monorepo |
| `npm run test`     | Run all tests (vitest) |
| `npm run build`    | Build client for production |

## Database

SQLite database stored at `server/data/musiclib.db`. Created automatically on first run with seed data.
